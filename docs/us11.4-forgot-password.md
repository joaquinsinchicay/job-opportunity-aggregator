# US11.4 - Forgot Password (solicitud de reseteo)

## Estado inicial (antes de los cambios)

### Criterios cumplidos
- Ninguno de los criterios de US11.4 estaba implementado de forma explícita.

### Criterios no cumplidos
- **US11.4.1**: en `/login` no existía el link `Forgot password?`.
- **US11.4.2**: no existía navegación a una pantalla de recuperación.
- **US11.4.3**: no existía pantalla `Forgot password` con campo `Email address` y botón `Send reset link`.
- **US11.4.4**: no existía validación de email inválido con el mensaje `Invalid email address`.
- **US11.4.5**: no existía solicitud de reset con mensaje de éxito `If the email exists, a reset link has been sent.`.
- **US11.4.6**: no existía manejo explícito de error de servicio con `Something went wrong. Please try again`.
- **US11.4.7**: no existía navegación de regreso a Login desde la pantalla Forgot password.

## Cambios implementados (mínimos y acotados a US11.4)

1. **Login: link de forgot password**
   - Se agregó el link `Forgot password?` en el formulario de login, apuntando a `/forgot-password`.

2. **Nueva pantalla Forgot Password**
   - Se creó `app/forgot-password/page.tsx` con el layout/card existente del flujo de autenticación.
   - Se agregó:
     - campo `Email address`
     - botón `Send reset link`
     - link `Back to Login` apuntando a `/login`

3. **Validación de email**
   - Se incorporó validación por regex en cliente.
   - Ante email inválido se muestra `Invalid email address`.

4. **Solicitud de reset a Supabase**
   - Se agregó `requestPasswordReset(email)` en `lib/supabase/auth.ts` usando endpoint `POST /auth/v1/recover`.
   - En submit exitoso del formulario se muestra `If the email exists, a reset link has been sent.`.

5. **Manejo de error de servicio**
   - Ante error en la solicitud se muestra `Something went wrong. Please try again`.
