# US11.2 - User Registration

## Estado inicial (antes de cambios)

### Criterios cumplidos
- **US11.2.1**: En Login ya existía el texto/link `Don't have an account? Sign up` hacia `/signup`.
- **US11.2.7 (parcial)**: Desde `/signup` ya existía navegación de regreso a Login mediante el link `Sign in`.

### Criterios no cumplidos
- **US11.2.2**: La pantalla `/signup` no tenía formulario con `Full name`, `Email address`, `Password` ni botón `Create account`.
- **US11.2.3**: No existía alta por email/password desde UI de registro.
- **US11.2.4**: No existían validaciones de nombre/email/password con mensajes requeridos.
- **US11.2.5**: No se mostraba el mensaje exacto `Email already registered` ante email duplicado.
- **US11.2.6**: No se mostraba el mensaje genérico `Something went wrong. Please try again` en errores de servicio del alta por email/password.
- **US11.2.8**: No existía redirección al Dashboard luego de registro exitoso por formulario.

## Cambios implementados

### Impacto
Se incorporó el flujo mínimo de registro por email/password en la pantalla existente de Sign up, preservando layout y estilos base del card/auth flow.

### Causa raíz
La página de alta estaba orientada únicamente a Google OAuth, sin soporte al formulario exigido por la US11.2.

### Archivos involucrados
- `app/signup/page.tsx`
- `lib/supabase/auth.ts`

### Detalle técnico
1. **Formulario de registro en `/signup`**
   - Se agregó formulario con campos `Full name`, `Email address`, `Password` y botón `Create account`.
   - Se mantuvo navegación de regreso a login (`Sign in` → `/login`).

2. **Validaciones de cliente**
   - `Full name` mínimo 2 caracteres → `Name must be at least 2 characters`.
   - Formato email inválido → `Invalid email address`.
   - `Password` mínimo 6 caracteres → `Password must be at least 6 characters`.

3. **Alta de cuenta y manejo de errores**
   - Se conectó el submit con `signUpWithPassword(fullName, email, password)`.
   - Se agregó envío de `full_name` en metadata del usuario hacia Supabase.
   - Se normalizó error de email duplicado a `Email already registered`.
   - Se normalizó error genérico de servicio a `Something went wrong. Please try again`.

4. **Post-registro exitoso**
   - Tras alta exitosa, se redirige a `/dashboard`.

## Validaciones ejecutadas

### Automatizadas
- `pnpm build` (OK)
- Script Playwright con validación de escenarios US11.2.1, 11.2.2, 11.2.3, 11.2.4, 11.2.5, 11.2.6, 11.2.8 (OK).
- Escenario US11.2.7 validado por navegación a URL `/login` desde el link `Sign in` de `/signup`.

### Observaciones
- `pnpm lint` falla por configuración base del repositorio (ESLint v9 sin `eslint.config.*`), no por cambios de esta US.

## Recomendación concreta
Mantener los mensajes de validación/error como constantes compartidas si en futuras US se agrega login por email/password, para evitar divergencia entre pantallas de autenticación.
