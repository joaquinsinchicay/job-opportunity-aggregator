# US11.5 - Reset password (completar desde link de email)

## Estado previo (antes de los cambios)

Evaluación de cumplimiento previa de criterios US11.5:

- **US11.5.1 Acceso a pantalla de reset**: **No cumplido**. No existía ruta/pantalla `/reset-password`.
- **US11.5.2 Formulario de nueva contraseña**: **No cumplido**. No existían campos `New password` / `Confirm password` ni botón `Reset password`.
- **US11.5.3 Password mínima**: **No cumplido**. No existía validación en flujo final de reset.
- **US11.5.4 Coincidencia de password**: **No cumplido**. No existía validación de confirmación en flujo final de reset.
- **US11.5.5 Reset exitoso**: **No cumplido**. No existía submit para actualizar contraseña ni redirección a `/login`.
- **US11.5.6 Link inválido o expirado**: **No cumplido**. No existía consumo del link/token de recovery ni manejo explícito de link inválido/expirado.

## Cambios implementados (mínimos para cumplir US11.5)

1. **Nueva pantalla de reset** (`/reset-password`)
   - Se creó `app/reset-password/page.tsx` respetando el layout/card actual de autenticación.
   - Incluye campos `New password`, `Confirm password` y botón `Reset password`.

2. **Consumo del link de recovery y sesión temporal**
   - Se agregó `consumePasswordRecoverySessionFromUrl()` en `lib/supabase/auth.ts` para:
     - leer tokens desde `window.location.hash`;
     - validar `type=recovery`;
     - guardar sesión temporal en storage;
     - detectar error de link inválido/expirado vía `error_description` y propagar error.

3. **Actualización real de password en Supabase**
   - Se agregó `updatePassword(password)` en `lib/supabase/auth.ts`, usando `PUT /auth/v1/user` con bearer token de recovery.

4. **Compatibilidad con forgot password existente (US11.4)**
   - Se ajustó `requestPasswordReset` para aceptar `redirect_to` opcional.
   - En `app/forgot-password/page.tsx` se envía `redirect_to` a `/reset-password` para que el link del email llegue a la nueva pantalla de US11.5.

5. **Mensajes y navegación solicitados**
   - Validación mínima: `Password must be at least 6 characters`.
   - Validación de match: `Passwords do not match`.
   - Link inválido/expirado: `Reset link is invalid or has expired`.
   - Éxito: `Password updated successfully` y redirección a `/login`.

## Impacto

- Se completa el flujo de recuperación iniciado en US11.4 sin alterar login/signup/google/logout.
- El impacto está acotado al flujo final de reset y a pasar `redirect_to` en el pedido de recover.

## Pruebas manuales sugeridas (US11.5)

1. Abrir link real de recovery recibido por email y verificar que abre `Reset password`.
2. Verificar presencia de `New password`, `Confirm password` y botón `Reset password`.
3. Ingresar password `< 6` y enviar: validar mensaje `Password must be at least 6 characters`.
4. Ingresar passwords distintos y enviar: validar mensaje `Passwords do not match`.
5. Ingresar passwords válidos y enviar: validar mensaje `Password updated successfully` y redirección a `/login`.
6. Abrir link inválido/expirado: validar mensaje `Reset link is invalid or has expired`.
