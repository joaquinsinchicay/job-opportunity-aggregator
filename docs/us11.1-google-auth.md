# US11.1 - Login con Google Auth

## Objetivo
Validar e implementar los ajustes mínimos necesarios para cumplir la User Story **US11.1** sin modificar UX/UI ni flujos de autenticación no relacionados.

## Estado inicial (antes de cambios)

### Criterios ya cumplidos
- **US11.1.1**: en `/login` ya se renderizaba el botón **"Continue with Google"**.
- **US11.1.2**: el botón de login llamaba `signInWithGoogleOAuth('/login')`.
- **US11.1.3**: al volver de OAuth con `#access_token` + `#refresh_token`, se guardaba sesión y se redirigía a `/dashboard`.
- **US11.1.4**: al cancelar OAuth, no se persistía sesión y el usuario permanecía en `/login`.
- **US11.1.5**: para usuario existente, el login OAuth completado redirigía a `/dashboard`.
- **US11.1.6**: para usuario nuevo, la creación/alta quedaba delegada en Supabase Auth al usar proveedor Google y, al completar OAuth, se redirigía a `/dashboard`.
- **US11.1.8**: si existe sesión activa (`getAccessToken()`), `/login` redirigía automáticamente a `/dashboard`.

### Criterio no cumplido / ajuste requerido
- **US11.1.7**: el mensaje de error de Google Auth en login no coincidía con el texto requerido. Existía el mensaje:
  - `Google sign in is currently unavailable. Please try again in a moment.`

## Cambios implementados

### 1) Mensaje de error alineado a US11.1.7
En `app/login/page.tsx` se actualizó el mensaje constante de error para Google Auth a:
- `Something went wrong. Please try again`

No se modificó layout, estilos, ni comportamiento de otros métodos de autenticación.

## Archivos modificados
- `app/login/page.tsx`
- `docs/us11.1-google-auth.md`

## Validaciones ejecutadas

### Validación técnica
- `npm run build` (ok)
- `npm run lint` (no ejecutable por configuración ESLint faltante en el repo)

### Validación funcional contra criterios Gherkin
1. **US11.1.1**: se confirmó visualmente botón "Continue with Google" en `/login`.
2. **US11.1.2**: revisión de código: click dispara `signInWithGoogleOAuth('/login')`.
3. **US11.1.3**: revisión de código: `consumeOAuthSessionFromUrlHash()` persiste sesión y redirige a `/dashboard`.
4. **US11.1.4**: revisión de código: cancelación OAuth sin tokens mantiene usuario en `/login`.
5. **US11.1.5**: revisión de flujo OAuth (Supabase) para usuario existente: redirección a `/dashboard`.
6. **US11.1.6**: revisión de flujo OAuth (Supabase) para usuario nuevo: alta automática por proveedor + redirección `/dashboard`.
7. **US11.1.7**: verificación del texto de error actualizado en login.
8. **US11.1.8**: revisión de código: sesión activa en `/login` dispara `router.replace('/dashboard')`.

## Impacto
- **Funcional**: cumplimiento explícito del mensaje requerido por US11.1.7.
- **Riesgo**: bajo; cambio acotado a una constante de UI en login.
- **Áreas afectadas**: únicamente pantalla `/login` para error en Google Auth.

## Recomendación
Mantener este documento como trazabilidad de cumplimiento de US11.1 y reutilizarlo como base para QA de autenticación OAuth.
